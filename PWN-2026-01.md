# Security Advisory: Ladybird JSON Parser UTF-8 Validation Denial of Service

**Advisory ID:** PWN-2026-001  
**Severity:** Medium (CVSS v3.1: 5.5)  
**Affected Component:** Ladybird JSON Parser  
**Issue Type:** Denial of Service (DoS)  
**Date Published:** April 27, 2026  
**Discoverer:** Paulino Calderon <paulino@calderonpale.com>

---

## Executive Summary

The Ladybird JSON parser contains a vulnerability in its UTF-8 validation logic that allows an attacker to trigger an illegal instruction exception (`SIGILL`) by providing specially crafted JSON input with invalid UTF-8 byte sequences. This causes the parser process to crash, resulting in a Denial of Service (DoS) condition.

The vulnerability affects any application using the Ladybird JSON parser to process untrusted JSON input without prior UTF-8 validation.

---

## Vulnerability Details

### Description

The Ladybird JSON parser fails to properly validate UTF-8 encoded input before processing JSON tokens. When the parser encounters invalid UTF-8 byte sequences (such as `0xcf`, `0xce`, `0xff`, `0x99`, `0xdf` in invalid positions), it attempts to process these bytes without validation, leading to an illegal instruction exception.

### Root Cause

The parser lacks an entry-point UTF-8 validation check. Instead of rejecting invalid UTF-8 with a descriptive error before parsing begins, the parser processes potentially malformed bytes through its parsing logic, triggering undefined behavior.

### Impact

- **Denial of Service (DoS):** Process crash via `SIGILL` signal
- **Stability:** Applications crash when processing untrusted JSON
- **Availability:** Service interruption if JSON parsing is critical to application operation
- **Scope:** Any Ladybird application processing untrusted JSON input

## Proof of Concept

### Crash Input (Minimal Case)

A 59-byte JSON payload with a single invalid UTF-8 byte causes `SIGILL`:

```json
{"­ey":"value","num":42,"arr":[1,2,3],"nested":{"a":true}}
```

**Hex representation (with invalid UTF-8 at offset 2):**
```
7b 22 cf 65 79 22 3a 22 76 61 6c 75 65 22 2c 22
6e 75 6d 22 3a 34 32 2c 22 61 72 72 22 3a 5b 31
2c 32 2c 33 5d 2c 22 6e 65 73 74 65 64 22 3a 7b
22 61 22 3a 74 72 75 65 7d 7d 0a
```

The byte `0xcf` at offset 2 is an incomplete UTF-8 sequence (valid start of a 2-byte UTF-8 character, but missing the continuation byte).

# Run parser with crash input

## Remediation

### For Developers Using Ladybird

#### Immediate Mitigation

1. **Input Validation:** Pre-validate all JSON input for UTF-8 compliance before parsing

```python
# Python example
import json

def safe_parse_json(data):
    """Parse JSON with UTF-8 pre-validation"""
    try:
        # Validate UTF-8 first
        if isinstance(data, bytes):
            data = data.decode('utf-8')
        
        # Then parse JSON
        return json.loads(data)
    except (UnicodeDecodeError, json.JSONDecodeError) as e:
        # Handle error gracefully instead of crashing
        print(f"Invalid input: {e}")
        return None
```

2. **Error Handling:** Wrap JSON parsing in try-catch to prevent process crashes

```javascript
// JavaScript example (if using Ladybird bindings)
function parseJSON(jsonString) {
    try {
        // Validate UTF-8
        new TextEncoder().encode(jsonString);
        
        // Parse JSON
        return JSON.parse(jsonString);
    } catch (error) {
        console.error("JSON parse error:", error.message);
        return null;
    }
}
```

#### Workaround

Until patches are available, sanitize JSON input:

```bash
#!/bin/bash
# Remove invalid UTF-8 bytes before parsing
sanitize_json() {
    local input_file="$1"
    local output_file="${input_file}.sanitized"
    
    # Remove invalid UTF-8 sequences
    iconv -f UTF-8 -t UTF-8 -c "$input_file" > "$output_file"
    
    echo "$output_file"
}

# Usage
SAFE_JSON=$(sanitize_json "untrusted_input.json")
/path/to/ladybird/parser < "$SAFE_JSON"
```

### For Ladybird Maintainers

**Recommended Fix:**

Add UTF-8 validation at the JSON parser entry point:

```cpp
// Pseudo-code fix
class JsonParser {
    bool parse(const std::string& input) {
        // Validate UTF-8 first
        if (!is_valid_utf8(input)) {
            return error("Invalid UTF-8 encoding");
        }
        
        // Then proceed with parsing
        return parse_tokens(input);
    }
    
private:
    bool is_valid_utf8(const std::string& s) {
        for (size_t i = 0; i < s.length(); ) {
            unsigned char c = s[i];
            
            if ((c & 0x80) == 0) {
                // ASCII (0xxxxxxx)
                i++;
            } else if ((c & 0xE0) == 0xC0) {
                // 2-byte (110xxxxx)
                if (i + 1 >= s.length() || (s[i+1] & 0xC0) != 0x80)
                    return false;
                i += 2;
            } else if ((c & 0xF0) == 0xE0) {
                // 3-byte (1110xxxx)
                if (i + 2 >= s.length() || 
                    (s[i+1] & 0xC0) != 0x80 || 
                    (s[i+2] & 0xC0) != 0x80)
                    return false;
                i += 3;
            } else if ((c & 0xF8) == 0xF0) {
                // 4-byte (11110xxx)
                if (i + 3 >= s.length() || 
                    (s[i+1] & 0xC0) != 0x80 || 
                    (s[i+2] & 0xC0) != 0x80 || 
                    (s[i+3] & 0xC0) != 0x80)
                    return false;
                i += 4;
            } else {
                return false;  // Invalid start byte
            }
        }
        return true;
    }
};
```
